Overview

Property types we can set:

  • 2D: _MainTex via texture=... and all others via textures={_Property=...}
  • Color: _Color via color=..., watertint=..., etc. and all others via shadercolors={_Property=...}
  • Float and Range(a,b): shadersettings={_Property=..., ...} and shadercolors={...={... , param="_Property", paramMin=a, paramMax=b}}
  • Vector: "Four component vectors and colors are the same in Unity shaders. (xyzw in the vector becomes rgba in the color)."

If you specify a shader that ends in ".shader" or ".txt" it will load that shader file from your skin folder. It does this by feeding the contents of the file into Unity's Material constructor so it must be a fixed function shader or one that was compiled by the Unity editor.

Textures can be loaded from JPG and PNG file formats. All textures will be compressed to DXT1 format (lossy) unless the path ends in ".png", in which case RGBA32 (lossless) format is used. The image loader is capable of recognizing the true file type even if the extension is wrong, so it should be possible to trick AS2 into loading PNGs with compression and vice versa by changing the filename. The Unity engine documentation implies that in this situation PNGs will be loaded with DXT5 and JPGs will be loaded with RGB24. Format descriptions are here in the "Desktop" section.
